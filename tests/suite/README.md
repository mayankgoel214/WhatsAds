# Autmn Test Suite

Comprehensive end-to-end testing framework for the V5 AI pipeline.

## Quick Start

```bash
cd /Users/lending/Autmn/tests/suite
pnpm install
# Ensure API server is running: pnpm dev:api from repo root
pnpm test                              # run all tests in manifest.json
pnpm test -- --product=diet-coke       # filter to one product
pnpm test -- --style=style_autmn_special  # filter to one style
pnpm test -- --dry-run                 # list what would run without executing
```

## Results

Each run creates a timestamped folder under `runs/`:
- `run.log` — streaming progress + event log (NDJSON)
- `outputs/` — all generated images, named `<product>_<style>_<instr>_run<N>.jpg`
- `results.csv` — structured test metadata (one row per style per API call)
- `results.json` — full data including prompts + analysis, keyed by test ID
- `grades.csv` — empty grade template; fill in manually after test completes

After grading:
```bash
pnpm report -- --run=<timestamp>
```

Generates `runs/<timestamp>/report.html` — open in browser. If `--run` is omitted, the most recent run is used.

## Manifest

Edit `manifest.json` to add products. Key fields:

- `styleTriplets`: array of 3-style arrays. The admin API accepts exactly 3 styles per call; each triplet is one API call.
- `instructions`: array of instruction variants. Use `"text": null` for no-instruction baseline.
- `runsPerCombo`: set to 2+ to test consistency within the same combination.

See full type definitions in `lib/types.ts`.

## Grading

After a run completes, `grades.csv` has one pre-filled row per successful generation with the test identity columns pre-filled. Fill in:

| Column | Description |
|---|---|
| `productAccuracy` | 0-100: Are the product's key physical features preserved? |
| `brandPreservation` | 0-100: Is branding/text readable and correct? |
| `styleMatch` | 0-100: Does output match the requested style? |
| `adQuality` | 0-100: Would this work as an actual ad? |
| `creativity` | 0-100: Is the composition creative and non-cliche? |
| `failureModes` | Comma-separated from FAILURE_MODES enum below |
| `observations` | Free text — anything worth noting |

Leave `weightedTotal` and `letterGrade` blank — `generate-report.ts` computes them from the 5 scores.

### Weights

Standard styles: productAccuracy 30%, brandPreservation 20%, styleMatch 20%, adQuality 20%, creativity 10%.

`style_autmn_special`: productAccuracy 25%, brandPreservation 10%, styleMatch 15%, adQuality 15%, creativity 35%.

## Failure mode catalog

From `lib/types.ts` `FAILURE_MODES` const:

| Mode | When to use |
|---|---|
| `PRODUCT_DISAPPEARED` | Product not visible in output |
| `PRODUCT_DAMAGED` | Product looks broken, distorted, melted |
| `PRODUCT_RECOLORED` | Product color changed significantly |
| `PIECE_MISSING` | Part of a set or multi-piece product is absent |
| `TEXT_HALLUCINATED` | Random text appears in background (not on product) |
| `BRAND_INCORRECT` | Brand text/logo is wrong or distorted |
| `ANGLE_WRONG` | Product angle/perspective changed undesirably |
| `CULTURALLY_OFF` | Setting or props are inappropriate for Indian market |
| `STYLE_MISMATCH` | Background/mood doesn't match requested style |
| `SCALE_WRONG` | Product is too small/large relative to scene |
| `BACKGROUND_ONLY` | Output is just a background with no product |
| `CLICHE_DEFAULT` | Generic/boring output (plain white, no creative choice) |
| `ANALYZER_FALLBACK` | Analyzer silently fell back to generic "product" label |

## File structure

```
tests/suite/
├── manifest.json          — product + config definitions
├── run-tests.ts           — main runner (pnpm test)
├── generate-report.ts     — HTML report generator (pnpm report)
├── lib/
│   ├── types.ts           — shared TypeScript types + grade helpers
│   ├── admin-client.ts    — admin API client with timeout + fallback detection
│   ├── logger.ts          — NDJSON streaming logger
│   └── concurrency.ts     — bounded parallel worker pool (no deps)
├── products/
│   ├── diet-coke/         — test product photos
│   └── kundan-polki/      — test product photos
└── runs/
    └── <timestamp>/       — created per run
        ├── run.log
        ├── results.csv
        ├── results.json
        ├── grades.csv
        ├── report.html    — generated by pnpm report
        └── outputs/       — downloaded images
```
